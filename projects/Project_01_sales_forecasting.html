Machine Learning

Sales Forecasting ML Pipeline

Predicting Revenue with 94% Accuracy

Role Data Engineer & ML Engineer
Stack Python, XGBoost, Pandas, Scikit-learn
Client E-Commerce Platform
Date 2024
Project Type Predictive Analytics
Overview

An end-to-end machine learning pipeline that ingests 3 years of transactional data, engineers over 60 features, and trains an XGBoost regression model to forecast weekly sales across 40+ product categories — achieving 94% accuracy and saving 12 hours of manual forecasting per week.

Python XGBoost Airflow Pandas Scikit-learn SQL
01

The Challenge

The client's merchandising team was manually forecasting sales using spreadsheets, leading to frequent stockouts and overstocking costing an estimated $2M annually. They needed an automated, reliable solution that could predict demand at both SKU and category level with confidence intervals.

02

The Solution

I designed a modular ETL pipeline using Pandas and SQLAlchemy to clean and version raw transactional data, then engineered lag features, rolling averages, and seasonal indicators. The XGBoost model was trained with time-series cross-validation to prevent data leakage. The final pipeline runs on a weekly Airflow DAG with automated retraining triggers when drift is detected.

03

The Outcome

94% MAPE accuracy on holdout test set. Inventory costs reduced by 18% in Q1 post-deployment. The model is now live in production serving forecasts to a React dashboard used by 6 merchandising analysts daily. Zero manual intervention required for 9+ months.
